1. 프로젝트 개요 분석
================================================================================

1-1. DolphinPod (Android 네이티브)
-----------------------------------
- 기술 스택: Kotlin + Jetpack Compose
- 플랫폼: Android 전용
- 아키텍처: MVVM 패턴, Room DB, WorkManager
- 데이터 수집:
  * UsageStatsManager (앱 사용량)
  * NotificationListenerService (알림)
  * CalendarSyncService (캘린더, TODO 상태)
  * SimpleQnA (일일 설문)
- 주요 기능:
  * 실시간 앱 사용량 모니터링
  * 알림 패턴 분석
  * 일별 스냅샷 생성
  * 리듬 점수 계산 (로컬)
  * AI 코멘트 (로컬 규칙 기반, GPT 연동 준비됨)
- UI 특징:
  * 바다 메타포 중심의 감정적 디자인
  * 시간대별 배경 변화 (아침/오후/저녁/밤)
  * 애니메이션 (파도, 돌고래, 버블 등)
  * Swipe Up으로 대시보드 이동
- 화면 구성:
  * Home (바다 메인 화면)
  * Dashboard (핵심 지표 카드)
  * DailyReport (하루 리포트)
  * SimpleQnA (일일 기록)
  * Settings (권한/설정)
  * Onboarding

1-2. Hibee (DPP - Web 기반)
----------------------------
- 기술 스택: React + TypeScript, Node.js + Express, Python FastAPI
- 플랫폼: 웹 브라우저
- 아키텍처: 프론트엔드/백엔드/AI 서버 분리
- 데이터 수집:
  * 시뮬레이션 데이터 (로컬스토리지 기반)
  * 실제 디바이스 데이터 수집 기능 없음
- 주요 기능:
  * 게이미피케이션 (레벨, 경험치, 스트릭)
  * Coral Score (0-100점)
  * AI 코멘트 및 제안 (FastAPI + OpenAI 연동 완료)
  * 일일 체크인 (DailyCheckIn)
  * 캘린더 뷰 (과거 리포트 조회)
  * 프로필 뷰 (레벨, 경험치, 통계)
- UI 특징:
  * Pixel Art 스타일
  * 바다 메타포 (OceanBackground 컴포넌트)
  * Glass morphism 효과
  * 반응형 디자인
- 화면 구성:
  * Onboarding
  * Dashboard (Coral Score, Ocean Log)
  * DailyCheckIn (기분/만족도/목표/메모)
  * CalendarView (과거 리포트)
  * ProfileView (레벨, 통계)
  * ReportView (상세 리포트)
- 백엔드 구조:
  * Node.js Express 서버 (포트 4000)
  * FastAPI AI 서버 (포트 8000)
  * OpenAI GPT-4o-mini 연동

2. 두 프로젝트의 강점 및 차이점 분석
================================================================================

2-1. DolphinPod의 강점
----------------------
✅ 실제 디바이스 데이터 수집 (Android 네이티브 API 활용)
✅ 실시간 모니터링 및 백그라운드 서비스
✅ 정교한 데이터 분석 로직 (시간대별, 카테고리별, 앱별)
✅ 안정적인 로컬 데이터 저장 (Room DB)
✅ 일별 스냅샷 자동 생성 (WorkManager)
✅ 완성도 높은 UI/UX (애니메이션, 시간대별 변화)

2-2. Hibee의 강점
------------------
✅ 게이미피케이션 시스템 (레벨, 경험치, 스트릭)
✅ AI 서버 연동 완료 (OpenAI GPT 기반)
✅ 백엔드/프론트엔드 분리 아키텍처
✅ 웹 기반 접근성 (크로스 플랫폼)
✅ Pixel Art 스타일의 독특한 디자인
✅ 체크인 시스템 (기분/만족도/목표)

2-3. 주요 차이점
----------------
| 항목 | DolphinPod | Hibee |
|------|-----------|-------|
| 플랫폼 | Android 네이티브 | 웹 (React) |
| 데이터 수집 | 실제 디바이스 API | 시뮬레이션 |
| AI 연동 | 로컬 규칙 기반 (GPT 준비) | FastAPI + OpenAI 완료 |
| 게이미피케이션 | 없음 | 레벨/경험치/스트릭 |
| 점수 시스템 | 리듬 점수 (0-100) | Coral Score (0-100) |
| 백엔드 | 없음 (로컬만) | Node.js + FastAPI |
| UI 스타일 | 부드러운 애니메이션 | Pixel Art |

3. 결합 전략: "Hibee 픽셀 아트 스타일 + DolphinPod 데이터 수집 기능 통합"
================================================================================

3-1. 핵심 원칙
--------------
1. UI는 Hibee의 픽셀 아트 스타일을 기반으로 통일
2. 바다 배경은 픽셀 아트 스타일로 구현 (OceanBackground 컴포넌트 기반)
3. DolphinPod의 강력한 데이터 수집 능력 유지 (Android 네이티브 API)
4. Hibee의 게이미피케이션 및 AI 서버 연동 통합
5. 픽셀 아트 디자인 시스템 일관성 유지
6. 단계적 마이그레이션 전략 수립

3-2. 아키텍처 설계
-------------------

[클라이언트 레이어]
┌─────────────────────────────────────────┐
│  DolphinPod Android App (Kotlin)        │
│  - UI: Jetpack Compose (바다 메타포)     │
│  - 데이터 수집: UsageStats, Notification│
│  - 로컬 저장: Room DB                   │
└─────────────────────────────────────────┘
                    │
                    │ HTTP/HTTPS
                    ▼
┌─────────────────────────────────────────┐
│  Node.js Backend (Express)              │
│  - API Gateway                          │
│  - 인증/인가                            │
│  - 데이터 동기화                         │
└─────────────────────────────────────────┘
                    │
                    │
                    ▼
┌─────────────────────────────────────────┐
│  FastAPI AI Server (Python)             │
│  - GPT-4o-mini 연동                     │
│  - 리포트 생성                           │
│  - 개인화 추천                           │
└─────────────────────────────────────────┘

3-3. 데이터 흐름
-----------------
1. Android 앱에서 실시간 데이터 수집
   → UsageStatsCollectorService
   → DolphinNotificationListenerService
   → SimpleQnA 입력

2. 로컬 저장 (Room DB)
   → AppSession, RawNotificationEvent
   → SimpleQnAResponse
   → DailySnapshot

3. 일별 스냅샷 생성 (자정 이후)
   → DailySnapshotWorker
   → DailySnapshot 생성

4. 서버 동기화 (선택적)
   → DailySnapshot을 서버로 전송
   → 게이미피케이션 데이터 동기화

5. AI 리포트 생성
   → 서버에 DailySnapshot 전송
   → FastAPI에서 GPT 호출
   → AI 코멘트 및 제안 수신

4-2. 게이미피케이션 시스템 통합
--------------------------------

[레벨 시스템]
- 초기 레벨: 1
- 경험치 획득: 일일 리포트 완성 시
- 경험치 계산: Coral Score 기반
  * 90점 이상: 50 EXP
  * 60-89점: 30 EXP
  * 30-59점: 15 EXP
  * 30점 미만: 5 EXP
- 레벨업 공식: level * 100 EXP 필요

[스트릭 시스템]
- 연속 달성 일수 추적
- 일일 리포트 완성 시 스트릭 증가
- 스트릭 보너스: 스트릭 * 2점 추가

[보상 시스템]
- 레벨업 시 특별 애니메이션
- 스트릭 달성 시 보너스 점수
- (향후) 뱃지 시스템 추가 가능

4-3. AI 서버 연동
-----------------

[기존 DolphinPod 로직]
- 현재: 로컬 규칙 기반 AI 코멘트
- GPTReport 모델/DAO는 이미 존재

[통합 전략]
1. DailySnapshot 생성 시 서버로 전송
2. 서버에서 FastAPI AI 서버 호출
3. GPT 기반 코멘트 및 제안 수신
4. 로컬 DB에 저장 (GPTReport)
5. UI에 표시

[API 엔드포인트 설계]
POST /api/ai/daily-report
Request:
{
  "user_id": "string",
  "date": "YYYY-MM-DD",
  "totalScore": number,
  "usage": {
    "totalTime": number,
    "lateNightTime": number,
    "longSessions": number,
    "shortFormRatio": number,
    "snsRatio": number,
    "gameRatio": number
  },
  "notifications": {
    "importantCount": number,
    "lowPriorityCount": number,
    "hasOverload": boolean
  },
  "checkIn": {
    "mood": number,
    "satisfaction": number,
    "goalAchieved": boolean,
    "memo": string
  },
  "profile": {
    "level": number,
    "experience": number,
    "currentStreak": number,
    ...
  }
}

Response:
{
  "comment": "string",
  "suggestion": "string"
}

4-4. 점수 시스템 통합
---------------------

[DolphinPod 리듬 점수 vs Hibee Coral Score]
- 두 점수 시스템의 로직 분석 필요
- 통합 방안: Coral Score를 메인으로 사용하되, 리듬 점수 개념 통합

[Coral Score 계산 로직 (Hibee 기준)]
- Base Score: 100점 만점
  * Screen Time: 20점 (목표 대비)
  * Late Night: 20점 (심야 사용 감점)
  * Long Sessions: 20점 (긴 세션 감점)
  * Short Form: 20점 (숏폼 비율 감점)
  * Check-in: 20점 (체크인 완료)
- Bonus Score: 스트릭 보너스
- Total Score: Base + Bonus

[통합 전략]
1. DolphinPod의 리듬 점수 계산 로직을 Coral Score 기준으로 재구성
2. SimpleQnA 응답을 Check-in 데이터로 매핑
3. 점수 Breakdown을 UI에 표시

5. 구현 단계별 로드맵
================================================================================

5-1. Phase 1: 기반 구축 (2주)

5-2. Phase 2: UI 통합 (3주)
---------------------------
□ Hibee 픽셀 아트 디자인 시스템 적용
  - Pixelify Sans, Press Start 2P 폰트 적용 (Android 폰트 리소스)
  - pixel-card, pixel-btn 스타일 컴포넌트 구현 (Compose)
  - 픽셀 그리드 배경 적용 (Canvas/Modifier.drawBehind)
  
□ OceanBackground 컴포넌트 구현 (픽셀 아트 스타일)

□ 모든 화면을 픽셀 아트 스타일로 리디자인

  - SimpleQnA/DailyCheckIn 화면 (픽셀 스타일 입력 폼)
  
□ Dashboard에 Coral Score 카드 추가 (픽셀 스타일)
□ 레벨/경험치 프로그레스 바 추가 (픽셀 스타일)
□ ProfileView 화면 구현 (픽셀 아트)
□ CalendarView 화면 구현 (픽셀 아트)
□ SimpleQnA를 DailyCheckIn 스타일로 개선 (픽셀 아트)

[Android Compose 픽셀 아트 구현 가이드]
- 폰트: assets/fonts/ 폴더에 PixelifySans, PressStart2P 추가
- 픽셀 그리드: Modifier.drawBehind로 Canvas에 그리드 그리기

- 픽셀 버튼: 클릭 시 translateX/Y 애니메이션
- image-rendering: Image의 Modifier에 적용 (필요 시)

-----------------
- Room DB 쿼리 최적화
- 네트워크 요청 최소화 (배치 처리)
- 이미지/애니메이션 최적화

6-4. 확장성
------------
- 향후 iOS 앱 개발 시 동일 API 사용
- 웹 버전과 데이터 공유 가능
- 멀티 디바이스 동기화 지원

6-5. Android Compose 픽셀 아트 구현
------------------------------------
- 폰트 리소스:
  * assets/fonts/pixelify_sans.ttf
  * assets/fonts/press_start_2p.ttf
  * FontFamily 정의 및 Typography 설정
  
- 픽셀 그리드 배경:

  * 8px x 8px 그리드 패턴
  * Canvas API 활용

  * @Composable fun PixelCard()
  * border = 4.dp
  * shadow = 6.dp offset

- 픽셀 버튼 컴포넌트:

  * 클릭 시 AnimatedVisibility/AnimatedContent
  * translateX/Y 애니메이션
  
- 바다 배경 컴포넌트:

  * 시간대별 색상 변화

7. UI/UX 통합 가이드라인
================================================================================

7-1. 디자인 원칙
-----------------
1. 픽셀 아트 스타일 일관성 유지 (Hibee 기반)
2. 바다 메타포를 픽셀 아트로 표현
3. 모든 UI 요소는 픽셀 아트 스타일로 통일
4. 게이미피케이션 요소 자연스럽게 통합
5. 시간대별 배경 변화 (픽셀 그라데이션)
6. 접근성 고려 (폰트 크기 조절, 명도 대비)

7-2. 픽셀 아트 디자인 시스템
------------------------------

[폰트]
- 기본 텍스트: Pixelify Sans
- 제목/버튼/큰 숫자: Press Start 2P
- image-rendering: pixelated 적용

[컴포넌트 스타일]
- .pixel-card:
  * border-4 border-[#2A5D7C]
  * shadow-[6px_6px_0px_0px_rgba(42,93,124,0.5)]
  * image-rendering: pixelated
  
- .pixel-btn-primary:
  * bg-[#3DA8C8] text-white
  * border-4 border-[#2A5D7C]
  * shadow-[6px_6px_0px_0px_rgba(0,0,0,1)]
  * hover/active 시 translate 효과
  
- .glass:
  * bg-white/30 backdrop-blur-sm
  * border-4 border-white/60
  * shadow-[4px_4px_0px_0px_rgba(0,0,0,0.2)]
  * image-rendering: pixelated

- .pixel-grid:
  * 8px x 8px 그리드 배경
  * rgba(255, 255, 255, 0.03) 선

7-3. 색상 팔레트
-----------------
- 바다 그라데이션 (Hibee 기반):
  * 상단: #87CEEB (하늘색)
  * 중상단: #5EC8E5 (밝은 파랑)
  * 중하단: #4FB8D8 (중간 파랑)
  * 하단: #3DA8C8 (진한 파랑)
  
- 강조 색상:
  * Primary: #3DA8C8 (PixelAqua)
  * Border: #2A5D7C (진한 파랑)
  * Success: #48C774 (레벨업/성공)
  * Warning: #FFB347
  * Error: #FF6B6B
  
- 텍스트:
  * Primary: hsl(210 85% 15%)
  * Secondary: hsl(199 40% 40%)

7-4. 바다 배경 (픽셀 아트)
---------------------------
- OceanBackground 컴포넌트 구조:
  * 픽셀 그리드 배경 (.pixel-grid)
  * 물결 효과 레이어 (wave-animate)
  * 버블 애니메이션 (bubble-rise)
  * 산호 장식 (하단 고정, float-animate)
  
- 시간대별 색상 변화:
  * 아침: 밝은 그라데이션 (#87CEEB → #5EC8E5)
  * 오후: 중간 그라데이션 (#5EC8E5 → #4FB8D8)
  * 저녁: 진한 그라데이션 (#4FB8D8 → #3DA8C8)
  * 밤: 어두운 그라데이션 (#3DA8C8 → #2A5D7C)

7-5. 애니메이션
---------------
- 픽셀 아트 스타일 애니메이션:
  * wave: 물결 효과 (6s ease-in-out infinite)
  * float: 부유 효과 (4s ease-in-out infinite)
  * bubble-rise: 버블 상승 (8s ease-in infinite)
  * blink: 깜빡임 효과 (2s ease-in-out infinite)
  
- 인터랙션 애니메이션:
  * 버튼 클릭: translate + shadow 변화 (0.1s)
  * 레벨업: 특별 픽셀 아트 애니메이션
  * 스트릭 달성: 축하 픽셀 아트 애니메이션

7-6. 이모지/아이콘
------------------
- 픽셀 아트 스타일 이모지 사용:
  * 🐬 (돌고래)
  * 🪸 (산호)
  * 🌿 (해초)
  * 🐠 (물고기)
  * 💧 (경험치)
  * 🌊 (바다)
  
- 필요 시 픽셀 아트 스타일 아이콘 이미지 제작 고려

11. 결론
================================================================================

이 전략 문서는 Hibee의 픽셀 아트 스타일과 바다 배경을 기반으로,
DolphinPod의 강력한 데이터 수집 능력을 통합하는 방안을 제시합니다.

핵심은:
1. UI는 Hibee의 픽셀 아트 스타일로 통일
2. 바다 배경은 픽셀 아트 스타일로 구현 (OceanBackground)
3. DolphinPod의 데이터 수집 기능 유지 (Android 네이티브 API)
4. Hibee의 게이미피케이션 및 AI 기능 통합
5. 단계적이고 안정적인 마이그레이션
6. 픽셀 아트 디자인 시스템 일관성 유지

이 전략을 통해:
- 독특하고 일관성 있는 픽셀 아트 스타일의 UI
- 강력한 데이터 수집 및 분석 기능
- 게이미피케이션을 통한 사용자 동기부여
- AI 기반 개인화된 리포트

이 모든 것을 결합한 통합 솔루션을 구축할 수 있습니다.
